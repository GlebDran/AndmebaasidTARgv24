Piirangud / ограничения

1. PRIMARY KEY - первичный ключ / уникальность набора полей (должен быть в каждой БД, не дает повторять значения)
2. FOREIGN KEY - вторичный ключ / связь с первичным ключом* (PK*) другой таблицы
3. NOT NULL - нельзя оставлять пустым
4. UNIQUE - что бы не было одинаковых данных, повторных - уникальный.
5. CHECK - выбор значений
-----------------------------------
SQL - Structure Query Language - язык запросв

DDL - Data Definition Language - язык определения данных - CREATE TABLE, ALTER TABLE, DROP TABLE - работа с полями
DML - Data Manipulation Language - язык управления данных - INSERT INTO, UPDATE, DELETE - работа с записями
